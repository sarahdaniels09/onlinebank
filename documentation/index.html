Installation

Follow these steps to get the project running locally.

Requirements

  • PHP 8+ with required extensions (OpenSSL, PDO, Mbstring, Tokenizer, XML).
  • Composer
  • MySQL or compatible RDBMS
  • Node.js & npm (for asset compilation)

Quick Setup

git clone <repo_url>
cd onlinebank
composer install
cp .env.example .env
php artisan key:generate
# configure .env (DB credentials etc.)
php artisan migrate --seed
npm install
npm run dev
php artisan serve

For production, configure a proper webserver (Nginx/Apache) and queue/cron workers.

Configuration

All secrets and environment-specific values must be set in the .env file. Example keys you'll need to set:

Application

APP_NAME=
APP_ENV=production
APP_URL=https://your-domain.com
APP_KEY=

Database

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_db
DB_USERNAME=your_user
DB_PASSWORD=your_password

Mail

MAIL_MAILER=smtp
MAIL_HOST=smtp.example.com
MAIL_PORT=587
MAIL_USERNAME=
MAIL_PASSWORD=

Payment

PAYSTACK_PUBLIC_KEY=
PAYSTACK_SECRET_KEY=
FLW_PUBLIC_KEY=
FLW_SECRET_KEY=
BITPAY_TOKEN=

Important: Never commit your .env file. Use the provided .env.example as a template.

cPanel (Shared Hosting) Installation

Use this when you host the site on a cPanel shared account. Two recommended approaches are Git deployment (cPanel Git Version Control) or uploading the archive and installing via SSH.

  1. Upload project files to your account. Recommended: upload a packaged ZIP file (CodeCanyon-style package) and extract it into a folder outside public_html.
  2. Install from ZIP (recommended for marketplace users):
    1. In cPanel > File Manager, click Upload and upload the ZIP package (for example onlinebank.zip).
    2. After upload, select the ZIP and click Extract. Extract to a folder such as onlinebank located in your home directory (outside public_html).
    3. Ensure the package includes a public folder. Move the contents of that public folder into public_html (or point the domain document root to the package's public folder if your host supports custom document roots).
    4. Rename .env.example to .env and update database and service credentials using File Manager's edit: specify DB host, name, user and password. If your package includes an SQL dump (e.g. database.sql), import it via phpMyAdmin or the cPanel MySQL Import tool.
    5. If the ZIP includes vendor/ and compiled frontend assets (public/css, public/js), you can skip running Composer and NPM on the server. If it does not, use SSH to run composer install and npm run prod when possible.
  3. Move the contents of the package public/ folder into public_html (or point the domain document root to the public folder if your host allows).
  4. Ensure the following directories are writable: storage and bootstrap/cache. Set CHMOD 755 or 775 as required.
  5. Create a MySQL database and user via cPanel > MySQL Databases and set those values in the .env file.
  6. If you have SSH access, run these commands over SSH in the project root: 
    composer install --no-dev --optimize-autoloader
cp .env.example .env
php artisan key:generate
php artisan migrate --force
php artisan db:seed --force
  7. If you don't have SSH/composer, install dependencies locally and upload the vendor/ folder. Make sure you match the server PHP version.
  8. Build frontend assets locally with npm run prod and upload the public/css and public/js files if the server can't run npm.
  9. Set up the scheduled tasks (cron) in cPanel: add a cron entry that runs every minute: 
    * * * * * php /home/username/path-to-project/artisan schedule:run >> /dev/null 2>&1
  10. For queues on shared hosting, use the database queue driver and run a cron that executes php artisan queue:work --sleep=3 --tries=3 or use a managed queue service.
  11. Configure SSL via cPanel's Let's Encrypt/AutoSSL or use the provider's SSL tool.

VPS (Ubuntu) Installation

Recommended for production. The example below uses Ubuntu, Nginx, PHP-FPM and MySQL.

  1. Update and install dependencies: 
    sudo apt update && sudo apt upgrade -y
sudo apt install nginx mysql-server php-fpm php-mbstring php-xml php-zip php-curl php-gd php-mysql unzip git curl -y
  2. Install Composer and Node.js (if needed): 
    curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

  4. Configure Nginx server block and point the root to /path/to/onlinebank/public. Example Nginx snippet: 
    server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    root /var/www/onlinebank/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.1-fpm.sock;
    }

    location ~ /\.ht {
        deny all;
    }
}
  5. Enable the site and reload Nginx: 
    sudo ln -s /etc/nginx/sites-available/onlinebank /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
  6. Set permissions: 
    sudo chown -R www-data:www-data /var/www/onlinebank
sudo find /var/www/onlinebank -type f -exec chmod 644 {} \;
sudo find /var/www/onlinebank -type d -exec chmod 755 {} \;
sudo chgrp -R www-data storage bootstrap/cache
sudo chmod -R ug+rwx storage bootstrap/cache
  7. Configure Supervisor for queue workers (install supervisor), example config: 
    [program:onlinebank-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/onlinebank/artisan queue:work --sleep=3 --tries=3
user=www-data
numprocs=1
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/var/log/onlinebank-worker.log
  8. Enable HTTPS with Certbot/Let's Encrypt: 
    sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
